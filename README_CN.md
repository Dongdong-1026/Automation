# 恒生指数双塔LSTM预测项目（接手文档）

## 0. 10分钟速读

本项目是一个面向恒生指数（`^HSI`）的时序预测系统，核心是**双塔模式感知LSTM（DualTowerPatternAwareLSTM）**，同时输出多周期收益率与波动率预测，并提供场景化回测与复盘能力。  
系统按 `Step0 -> Step1 -> Step1.5 -> Step2 -> Step3 -> Step6 -> Step8 -> Step9` 顺序执行，所有关键产物沉淀在 `model_artifacts/`，用于后续复现、审计、推理和运维。

---

## 1. 项目整体思路

### 1.1 核心目标与设计初衷

- **核心目标**：对 `^HSI` 进行**多预测期（1/5/10/15/20/25/30天）收益率**与**未来波动率**联合建模，服务于择时、风险提示和趋势跟踪。
- **设计初衷**：
  - 解决单一短期预测易受噪音干扰的问题。
  - 解决仅做价格点预测、无法量化不确定性的问题。
  - 解决历史回测中常见的数据泄露问题（通过时序切分+场景重训做审计隔离）。

### 1.2 整体架构设计（模块划分与关联逻辑）

#### 模块A：数据接入与缓存（Step0）
- 数据源：`yfinance` 为主，失败时可走兜底源；本地缓存目录 `yf_cache/`。
- 目标：稳定获得 OHLCV 与外部联动资产数据，减少重复拉取。

#### 模块B：因子工程（Step1）
- 核心函数：`detect_trading_patterns(...)`。
- 目标：构建技术、量价、波动率、宏观、跨市场等多类候选因子。

#### 模块C：特征筛选（Step1.5）
- 两阶段筛选：**MI 初筛 -> 相关性剪枝 -> PI 再筛**。
- 输出：`selected_features*.json`，用于后续训练/推理特征对齐。

#### 模块D：数据预处理（Step2）
- 时序切分、标准化、滑窗序列化、标签构建（7收益率+1波动率）。
- 输出：模型输入张量及 scaler 相关产物。

#### 模块E：模型训练与调参（Step3.1/3.2/3.3）
- 模型：`DualTowerPatternAwareLSTM`（特征注意力 + 时序编码 + 多头输出）。
- 可选 HPO：Optuna。
- 输出：`best_model.pth`、配置文件、训练指标。

#### 模块F：推理与结果输出（Step6/Step8）
- 基于已训练模型进行自回归多步推理。
- 输出预测路径、分位区间、决策表与文本报告。

#### 模块G：场景回测与复盘（Step9 + review脚本）
- Step9 walk-forward 场景重训，避免前视偏差。
- `step8_review.py` / `daily_1d_review.py` 用于到期复盘与命中率统计。

### 1.3 技术选型依据（与需求匹配）

- **PyTorch**：适配多任务时序建模，便于实现双塔结构与多输出损失组合。
- **Pandas/Numpy**：高效完成时序特征构建与窗口计算。
- **Scikit-learn**：标准化、MI、随机森林与 PI 等传统特征筛选能力成熟。
- **Optuna**：自动搜索超参数，降低人工试参成本。
- **Jupyter Notebook + 脚本化执行**：兼顾研究迭代灵活性与自动化运行需求。
- **本地缓存机制**：提升日常运行稳定性和效率，减少外部接口波动影响。

### 1.4 核心价值与应用场景

- **核心价值**
  - 同时给出方向与波动视角，支持收益-风险联合判断。
  - 通过场景重训回测提升评估可信度。
  - 产物结构化沉淀，便于交接、复现、审计。
- **应用场景**
  - 日度择时与仓位管理。
  - 中短期趋势研判与风险预警。
  - 模型迭代对比和策略复盘。

## 2. 因子大致情况

> 本节按“定义/作用/来源/生成方式/参数/场景/关联关系”说明核心因子族。

### 2.1 因子总览

- 因子分层：**宏观因子、技术因子、量价因子、波动率因子、相关性因子、跨市场因子**。
- 常用窗口参数：`5`、`30`、`60`、`252` 交易日（对应短中长与年度统计视角）。
- 因子筛选后得到稳定特征子集，并由 `selected_features*.json` 管理。

### 2.2 核心因子清单（代表项）

#### A. 宏观/跨市场因子

- **`US10Y_Close`**
  - 定义：10Y美债收益率水平。
  - 作用：刻画全球利率环境与估值压力。
  - 来源：外部指数数据（`^TNX`）。
  - 场景：中长期趋势判断、估值敏感阶段。

- **`USDCNH_Close` / `CNH_Change_5D`**
  - 定义：离岸人民币水平及短期变动。
  - 作用：反映汇率压力与资金流向。
  - 来源：`CNH=F`。
  - 场景：风险偏好变化、南北向资金扰动期。

- **`VIX_Quantile`**
  - 定义：VIX 在滚动窗口中的分位。
  - 作用：量化市场恐慌程度。
  - 来源：`^VIX`。
  - 参数：通常用 `252` 日窗口。
  - 场景：短期风险预警与波动率放大阶段。

- **`ES_SPX_Basis`**
  - 定义：美股期现结构差异指标。
  - 作用：衡量风险偏好与预期变化。
  - 场景：跨市场联动强化阶段。

#### B. 技术/动量因子

- **`BB_Pos`**
  - 定义：价格在布林带内的相对位置。
  - 作用：识别超买超卖与均值回归压力。
  - 生成：基于滚动均值与标准差计算上下轨后归一化。

- **`RSI_Velocity`**
  - 定义：RSI 的变化速度。
  - 作用：捕捉动量拐点和短期情绪变化。

- **`MACD_Bearish_Cross`**
  - 定义：MACD 空头交叉事件。
  - 作用：趋势反转信号增强。

- **`Returns_ZScore`**
  - 定义：收益率经波动尺度化后的标准分。
  - 作用：提升不同波动状态下的可比性。

#### C. 量价/波动率因子

- **`Vol_Rel_Short` / `Vol_Rel_5`**
  - 定义：当前成交量相对短期均值的比例。
  - 作用：识别放量突破/衰减。

- **`Volatility_Cluster`**
  - 定义：波动聚集程度。
  - 作用：描述中期路径不确定性。

- **`BB_Width_Compressed`**
  - 定义：布林带宽度压缩状态。
  - 作用：提示潜在方向性突破前状态。

#### D. 相关性因子

- **`Beta_SPX`**
  - 定义：对 SPX 的暴露系数。
  - 作用：衡量海外市场冲击传导强度。

- **`Correlation_VIX`**
  - 定义：与 VIX 的滚动相关性。
  - 作用：风险偏好阶段性变化刻画。

### 2.3 因子获取/生成方式

- 数据拉取与缓存：先读 `yf_cache/`，不足部分增量更新。
- 缺失处理：对关键字段做对齐与兜底填充。
- 因子生成：在主数据表基础上按窗口计算技术、统计、跨市场联动特征。
- 特征筛选：MI + PI 与相关性去冗余结合，平衡信息量与稳定性。

### 2.4 因子间关联关系（接手重点）

- **短期层（1D/5D）**：更依赖情绪与技术因子（如 `VIX_Quantile`、`RSI_Velocity`）。
- **中期层（10D/20D）**：更依赖波动结构与相关性因子（如 `Volatility_Cluster`、`Beta_SPX`）。
- **长期层（25D/30D）**：更依赖宏观与汇率联动（如 `US10Y_Close`、`USDCNH_Close`）。
- 因子并非静态有效：重要性会随市场状态变化，需结合回测与复盘持续审视。

## 3. 项目使用方法

### 3.1 环境搭建

#### 3.1.1 运行环境建议
- Python 3.9+（建议统一团队版本，避免依赖差异）。
- 具备 `pip` 与 Jupyter 运行能力。

#### 3.1.2 依赖安装

```bash
pip install torch pandas numpy scikit-learn yfinance TA-Lib optuna matplotlib seaborn scipy
```

> 备注：`TA-Lib` 在不同系统安装方式有差异，若失败请先安装系统级 `ta-lib` 再执行 `pip install TA-Lib`。

#### 3.1.3 目录与关键文件
- 主流程：`LSTM_twotarget_v3.ipynb`
- 复盘脚本：`step8_review.py`、`daily_1d_review.py`
- 报告参考：`HSI_Model_Trend_Analysis_Report.md`
- 产物根目录：`model_artifacts/`

#### 3.1.4 环境变量（自动执行常用）

- **`RUN_AUTO`**：是否自动模式（建议 `1`）。
- **`TICKER`**：标的（如 `^HSI`）。
- **`START` / `END`**：训练区间。
- **`INCLUDE_TODAY`**：是否纳入当日。
- **`RUN_FS`**：是否执行特征筛选。
- **`RUN_HPO`**：是否执行超参优化。
- **`VOL_STRATEGY` / `VOL_VALUE`**：成交量兜底策略与数值。

#### 3.1.5 关键输入参数模板（日期范围/缺失值兜底）

**A. 日期范围输入（必填优先项）**
- **`START`**：起始日期，格式 `YYYY-MM-DD`，建议覆盖至少 5 年交易日。
- **`END`**：结束日期；留空表示取最新可得数据。
- **`INCLUDE_TODAY`**：`Y/N`，控制是否纳入当日数据（当日数据不完整时建议 `N`）。

**B. 缺失成交量输入（实盘常用）**
- **`VOL_STRATEGY=manual`** + **`VOL_VALUE=<纯数字>`**。
- 当数据源未返回当天成交量时，使用该值兜底，避免流程中断。

**C. 缺失波动率输入（接手必读）**
- 本项目不要求手工输入波动率数值；波动率目标由历史价格序列在流程内重建。
- 若出现“波动率列缺失/为空”，建议按以下顺序处理：
  1. 先补齐 OHLC 基础行情并重跑 Step0/Step1；
  2. 在 Step2 重新生成波动率相关特征与目标；
  3. 检查 `run_metrics.json` 中波动率相关指标是否恢复正常。
- 若必须人工兜底，建议仅在临时排障使用短窗口历史波动估计值，并在运行日志中记录兜底来源。

### 3.2 启动方式

#### 3.2.1 本地交互运行（开发调试）
1. 打开 `LSTM_twotarget_v3.ipynb`。
2. 按 Step 顺序执行核心单元（建议全链路）。
3. 检查 `model_artifacts/` 中新增 run 目录。

#### 3.2.2 参数输入方式（Notebook 参数区）

在 `LSTM_twotarget_v3.ipynb` 的参数配置区域填写以下输入：

- **固定历史区间回放**
  - `RUN_AUTO=1`
  - `TICKER="^HSI"`
  - `START="2015-01-01"`
  - `END="2024-12-31"`
  - `INCLUDE_TODAY="N"`
  - `VOL_STRATEGY="manual"`
  - `VOL_VALUE="3500000000"`
  - `RUN_FS=0`
  - `RUN_HPO=0`

- **日常增量运行（END留空）**
  - `RUN_AUTO=1`
  - `TICKER="^HSI"`
  - `START="2006-01-01"`
  - `END=""`
  - `INCLUDE_TODAY="Y"`
  - `VOL_STRATEGY="manual"`
  - `VOL_VALUE="12345678"`
  - `RUN_FS=0`
  - `RUN_HPO=0`

#### 3.2.3 复盘运行

```bash
python daily_1d_review.py
```

该命令会触发 `step8_review.py`，生成 Step8 到期复盘汇总表。

### 3.3 核心功能使用步骤

#### 功能A：训练与模型产物
1. 执行 Step0~Step3。
2. 确认生成：
   - **`best_model.pth`**
   - **`best_hyperparameters.json`**
   - **`selected_features*.json`**
   - **`run_metrics.json`**

#### 功能B：推理与决策输出
1. 执行 Step6 与 Step8。
2. 查看预测路径、分位区间、比例化输出报告。

#### 功能C：场景回测与审计
1. 执行 Step9（walk-forward）。
2. 检查不同保留窗口场景下指标差异，重点观察 MAE 与方向准确率变化。

### 3.4 关键配置与参数说明

- **序列长度**：`SEQUENCE_LENGTH=30`
- **预测期**：`[1,5,10,15,20,25,30]`
- **训练轮数/早停**：`epochs=200`，`patience=15`
- **批大小**：`batch_size=64`
- **损失结构**：收益率 + 波动率 + 方向损失加权

### 3.5 常见问题与解决方案

#### 问题1：特征文件缺失（`selected_features*.json not found`）
- 原因：未执行筛选或路径配置错误。
- 处理：
  - 将 **`RUN_FS=1`** 重新生成；
  - 或指定历史稳定特征文件路径。

#### 问题2：维度不匹配（如输入维度与模型不一致）
- 原因：训练与推理阶段使用的特征清单不同。
- 处理：
  - 强制按 `selected_features_applied.json` 对齐列顺序与数量；
  - 必要时重跑 Step1.5 + Step2。

#### 问题3：数据拉取失败或当日成交量缺失
- 原因：外部数据接口波动。
- 处理：
  - 使用缓存数据；
  - 设置 **`VOL_STRATEGY=manual`** 并提供 **`VOL_VALUE`**。

#### 问题4：波动率相关列缺失或预测异常
- 原因：OHLC 不完整、滚动窗口样本不足、或前序步骤未重算波动率特征。
- 处理：
  - 确认原始行情列完整（至少 Open/High/Low/Close）；
  - 扩大日期区间（建议增加 `START` 历史长度）后重跑 Step0~Step2；
  - 检查输出指标和日志，确认波动率头训练已恢复。

#### 问题5：Notebook 参数修改后未生效
- 原因：参数单元未重新执行，或执行顺序被打乱。
- 处理：
  - 修改参数后先单独执行参数单元；
  - 再按 Step 顺序全链路执行关键单元。

### 3.6 接手维护建议

- 保持 `model_artifacts/` 的 run 目录完整，避免只拷贝权重不拷贝配置。
- 每次改动因子工程后，必须重新校验特征筛选与推理对齐链路。
- 定期执行复盘脚本，跟踪模型实际命中率和漂移迹象。
